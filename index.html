About OSC

Oscillate [OSC] is an ASIC resistant fork of TurtleCoin, using the CryptoNight_Turtle algorithm. Quick transactions, privacy, Ease of use make this coin perfect for paying anyone, anywhere. Oscillate can also be mined on any windows or Linux based PC, no matter how strong or weak!


If you do not know what TurtleCoin is, it is a fairly popular cryptonote coin, which is what this was forked from. Oscillate was forked from this code for a few reasons.


  • 1. The code is updated frequently and is always supporting updates for operating systems.

  • 2. The code is lightweight and can be used on ARM devices.

  • 3. It is faster than the original cryptonote commit.

  • 4. An easy to use wallet is included with the release.

Oscilalte will be forking to a custom algorithm called Hydra Chaos.


  • HydraChaos is completely built from the ground up. While it can merge other coins into the algo easily... it can only do so because of the parent algo.. once a coin adopts HC it's very hard to fork away from it. The design behind the algo is 3 fold logic. First we want to ensure that no single person can manipulate the network. With HC there is no such thing as a 51% attack. As the algo requires 2 chains that link with microchains. On top of that, HC mutates and verifies transactions with push design instead of a pull design. All algos out there currently use the pull design. They ask the daemon for information and permission to create a transaction... with HC the daemon and chains watch the potential transaction pool and push the information that it validates. The miners have very little control in that regard. In fact, it is entirely possible that a mining program could be open and never hash a single validation (unlikely but possible) and still be rewarded due to the nature of RoundRush. RoundRush is a new technique developed specifically for HC that pushes hashing info to available miners. The miners arent able to do anything until the daemon says so. As the algo mutates there is no possible way for the miner to know what the algo has mutated to without direct input from the daemon
